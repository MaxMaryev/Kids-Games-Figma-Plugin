# 😊 Kids Games Plugin 😊

Плагин для [Figma](https://www.figma.com/) с двумя инструментами для подготовки игровой графики:

1. **Растр** — превращает выделенный слой или группу в PNG прямо на канвасе. Появляется прямоугольник с картинкой-заливкой — удобно, когда нужно «зафиксировать» вектор и работать с ним как с готовым ассетом, не выходя из файла.
2. **Кратность 4** — проверяет, что ширина и высота слоя кратны четырём (важно для текстурных атласов, сжатия и платформенных требований). Если кратности нет, плагин аккуратно оборачивает слой фреймом нужного размера. Сама графика при этом не растягивается.

## Требования

- [Node.js](https://nodejs.org/) (LTS) — для сборки.
- [Figma для десктопа](https://www.figma.com/downloads/) — плагины из режима разработки запускаются только в десктопном клиенте; в браузерной версии этого пункта меню нет.

## Сборка

```bash
npm install
npm run build
```

Скрипт [`build.cjs`](build.cjs) собирает [`src/main.ts`](src/main.ts) в `code.js` и встраивает [`ui.html`](ui.html) в плейсхолдер `__html__` (как ожидает [`manifest.json`](manifest.json)).

Для разработки с пересборкой при изменениях:

```bash
npm run watch
```

## Подключение в Figma

1. Соберите проект (`npm run build`) — рядом должен лежать актуальный `code.js`.
2. В десктопной Figma: **Plugins** → **Development** → **Import plugin from manifest…**.
3. Укажите `manifest.json` из корня проекта.
4. Запуск: **Plugins** → **Development** → **😊 Kids Games Plugin 😊**.

Подробности по полям манифеста — в [документации Figma Plugin API](https://developers.figma.com/docs/plugins/manifest).

## Вкладка «Растр»

1. Выделите один или несколько слоёв.
2. Выберите масштаб (1×–4×) — тот же множитель, что и при обычном экспорте PNG.
3. По желанию поставьте галочку:
   - **Заменить оригинал** — исходный слой удаляется, остаётся только растр.
   - **Скрыть оригинал** — исходный слой остаётся, но становится невидимым.
4. Нажмите **В растр**.

На месте оригинала появляется прямоугольник с PNG-заливкой; выделение переходит на новые растры — их сразу можно подвинуть или проверить. Если родитель с auto layout, растр позиционируется абсолютно, чтобы встать ровно туда, где был оригинал, и не сдвинуть соседей в ряду.

## Вкладка «Кратность 4»

Два действия — **Проверить** и **Поправить**.

### Проверить

Смотрит на размеры с учётом теней, обводок и размытий — то есть на то, сколько места слой реально занимает на экране. Размеры округляются вверх до целых пикселей, потом проверяется кратность 4.

В выделении остаются только нарушители; в списке внизу появляются их имена, текущие размеры и до чего нужно дотянуть. Клик по строке (или по «Показать») переводит фокус Figma на слой.

### Поправить

Создаёт фрейм-обёртку вокруг слоя:

- Обёртка получает имя исходного слоя — в дереве слоёв «контейнер» выглядит знакомо.
- Сам слой переезжает внутрь и переименовывается в **`[no4] <старое имя>`** — так видно, какие слои изначально не проходили проверку.
- Размер обёртки увеличивается до ближайшего сверху кратного 4 по каждой стороне (не больше +3 пикселей). Графика внутри не растягивается: меняется только контейнер, содержимое смещается так, чтобы визуально остаться на месте.
- После операции в выделении — сами обёртки.

Чего **Поправить** не делает:

- Не трогает слои, у которых размеры уже кратны 4.
- Не оборачивает в саму страницу/документ или в главный компонент — это могло бы поломать инстансы.
- В редких случаях, когда слой сильно обрезается родителем, проверка может разойтись с фактическим PNG при экспорте. Это компромисс между точностью и скоростью; на типичных ассетах не проявляется.

## Ограничения

- Нужно явное выделение — без него плагин напомнит выбрать слой.
- Для растра у слоя должны быть валидные границы.
- Плагин не выходит в интернет — все операции локальные.

## Стек

- TypeScript, сборка [esbuild](https://esbuild.github.io/), типы [`@figma/plugin-typings`](https://www.figma.com/plugin-docs/).

## Структура исходников

- [`src/main.ts`](src/main.ts) — `showUI`, маршрутизация сообщений.
- [`src/features/rasterize.ts`](src/features/rasterize.ts) — растрирование.
- [`src/features/multipleOfFourCheck.ts`](src/features/multipleOfFourCheck.ts) / [`multipleOfFourFix.ts`](src/features/multipleOfFourFix.ts) — Проверить / Поправить.
- [`src/domain/multipleOfFour.ts`](src/domain/multipleOfFour.ts) — чистая математика кратности и отступов.
- [`src/figma/geometry.ts`](src/figma/geometry.ts), [`nodeQueries.ts`](src/figma/nodeQueries.ts) — геометрия и запросы к узлам.
